客户拥有什么
每个代币是 SPV 持有对应底层资产的 1:1 受益权。SPV 破产隔离——即使平台运营公司倒闭，资产仍为代币持有人保留。这一点跟 FTX 截然不同：FTX 客户资产跟运营资金混同，所以平台一倒所有客户钱都没了；Oria 的客户资产法律上不属于运营公司，运营公司倒闭跟客户无关。
服务商配置
独立基金管理人负责 NAV 计算和投资人名册——独立第三方做账，平台无法单方面操纵账目。四大会计师事务所签署年度财务报告。开曼监管要求 SPV 董事会含独立董事，独立董事不能是平台员工或关联方。
税务效率
开曼 SPV 在实体层面税务中性。税务义务在投资人本国司法辖区。SPV 内部撮合（用户买卖订单先内部对冲，再剩余净额走市场）减少交易频次和经纪佣金——这是基金运营的标准做法，不是规避。规范说法叫"税务效率"，不叫"避税"——前者是合法的法律结构优化，后者是违法的逃税。
给股东的关键说明
这套结构是 RWA 行业的标准操作（Ondo、Securitize、Backed Finance 都用类似模式），监管已经接受。Cayman + SG/HK 的组合解决了三件事：发行灵活性、银行账户开立、机构信誉。BVI 替代开曼成本低 30%，但机构投资人认可度略差，建议成熟期再考虑。

07 代币与清算 技术

ERC-3643 标准，多链 USDT 接收，强制白名单。

代币标准
ERC-3643（T-REX 标准）。原生链上身份注册表，转账限制在 solidity 层强制执行，每个资产可挂载独立合规模块。证券型代币的行业标准——Securitize、Tokeny 等头部玩家在用。这意味着我们不重新发明轮子，而是站在已经被监管接受的标准上。
清算层
首发与赎回在许可链或 L2（费率低、终结性快）。跨链桥支持有需要的用户提到以太坊主网。USDT 多链接收（以太坊、Tron、Solana、Arbitrum）——对用户透明。用户在哪条链有 USDT 都能用，不需要先做跨链。这是降低开户门槛的关键技术决策。
合约逻辑
入金确认即铸币，赎回请求即销毁。转账要求双方地址都在白名单——非 KYC 地址无法接收。即使用户把代币提到自己钱包，再想转给朋友，朋友必须先完成 KYC 才能收到。这是合规的硬约束，监管能接受、客户能理解。暂停权限由独立董事加 MLRO 多签持有，仅在监管要求时使用。
审计安排
上线前两家独立机构审计（Trail of Bits、OpenZeppelin、Certora 或同级）。开通漏洞赏金。任何合约升级触发年度复审。这一点对老钱客户是隐性的信任信号——他们不一定看得懂代码，但他们认得 Trail of Bits 这个品牌。

09 资产质押消费 支付

不卖资产也能花钱——把私行级证券质押贷款带到链上。

客户价值
用户持有的代币化资产可以作为抵押品，瞬时获得 USDT 信贷额度用于消费——不需要卖出资产、不触发税务事件、不打乱投资组合。私行客户最熟悉的服务（高盛 PWM、瑞银 Lombard Lending），现在用 USDT 结算 + 智能合约实现，几秒钟到账。

典型场景：客户持有 100 万美债代币，临时要付 20 万美元的孩子学费——传统路径要赎回美债（损失利息+市价波动）、提现到银行（T+2）、再汇款（手续费+时间）；Oria 路径是抵押 30 万美债代币、即时拿 20 万 USDT、刷卡或转账完成。资产不动，钱已经花了。

利率与定价
浮动利率，按 SOFR + 风险溢价 + 平台利差计算。典型水平 6-9% APR，比信用卡循环利率（18-25%）低得多，比私行 Lombard Lending（5-7%）略高一些——但 Oria 的优势是开户瞬时到账，私行 Lombard 要走 1-2 周授信流程。利率每月透明披露在 Dashboard。
合规结构
在开曼或 BVI 设立独立 Lending SPV，持有放贷牌照（Cayman MS Act / BVI Financing Act）。SPV 资金来源：股东资本 + 机构信贷额度（与对冲基金、家族办公室合作）。不动用客户资产做杠杆——这是 FTX/Celsius 暴雷的根本原因，平台不踩这条红线。

抵押品的链上锁定由智能合约自动执行，清算逻辑公开可审计：当抵押品价值跌破清算阈值，合约自动卖出抵押资产偿还贷款，剩余返还用户钱包。这套机制比传统券商的"保证金电话 + 手动清算"更快、更透明。
给股东的关键说明
质押消费功能不仅是产品差异化，更是平台第二大收入引擎。私行客户的 Lombard Lending 余额通常占 AUM 的 15-25%。如果 Oria 达到 5 亿美元 AUM、质押率 20%、净利差 4%，仅这一项业务年收入约 400 万美元，且毛利率高于 80%。这是私行业务模式的核心利润来源——我们要复制它。

11 MCP Server 与开放 API Agent Native

让任何 LLM 和 agent 框架都能直接调用 Oria——零定制开发。

为什么需要
2024 年底 Anthropic 发布的 MCP（Model Context Protocol）已经成为 LLM 与外部系统连接的事实标准——Claude、ChatGPT、Gemini 都在支持。这意味着任何接入 MCP 的 agent 都能"看到"所有暴露 MCP 接口的服务。

Oria 在 2026 年早期把全部业务能力包装成 MCP Server，把整个平台变成一个 agent 可调用的金融工具——不需要任何 agent 厂商专门为我们做集成，他们的用户开箱即用就能让 agent 操作 Oria。
能力清单
所有人类能用 App 做的事，agent 都能通过 API 做：查询持仓与净值、提交买卖订单、申请质押额度、发起 USDT 出入金、Visa 卡消费授权、配置自动再平衡规则、获取税务报表与交易历史。

12 Agent 身份与权限 Agent Native

把 agent 当独立法律主体来管理，而不是人类的延伸账号。

核心问题
让 agent 用人类的账号登录是错误的。agent 会犯错、会被劫持、会执行幻觉指令——如果它登录的是人类主账户，一次失误可能清空全部资产。

正确的做法：每个 agent 是独立身份，绑定到人类主账户下，权限严格分级。这是借鉴企业 IAM（身份与访问管理）的成熟模式——CFO 不会让公司每个员工都用 root 账号，agent 也不应该。

紧急控制
人类主账户随时可一键吊销 agent——不需要联系客服、不需要等待。所有 agent 操作实时推送到人类主账户的告警通道（邮件 + Telegram + iOS push），异常行为可手动 rollback。

合规层面，每个 agent 有独立的链上身份（DID），所有操作上链留痕，监管和审计可追溯到具体 agent 而不只是主账户。出问题时责任划分清晰：是人类授权过度、还是 agent 越权——链上证据自动判定。
13 意图驱动的执行引擎 Agent Native

从"做什么"到"达成什么"——把意图翻译成最优执行路径。

范式转变
传统 API · 命令式
告诉系统具体怎么做
POST /order
{ asset: "dSPY",
  side: "buy",
  amount: 100 }
Oria Intent API · 意图式
告诉系统要达成什么
POST /intent
{ goal: "保持 50/40/10 配置,
  把月度结余配置进去",
  constraints: {...} }
为什么 agent 时代必须这么做
命令式 API 要求调用方知道每一步该怎么做——这对 agent 来说意味着每次都要 LLM 推理出完整执行路径，慢、贵、容易出错。意图式 API 让 agent 只表达"我要什么"，具体路径优化由 Oria 内置规则引擎处理：成本最低的资产路径、最优的税务时点、滑点保护、价格区间限制——这些是 Oria 比 agent 更专业的领域。

这是一种责任划分：agent 负责理解客户意图，Oria 负责把意图翻译成最优金融执行。两边各做自己最擅长的，整个系统才稳定。
典型意图
"每月发薪日预留 50 万 USDT，剩余配置进美债生息"——周期性任务，agent 设置一次后自动执行。

"当组合美股仓位超过 45% 时，把超出部分换成黄金"——条件触发任务，市场波动时自动响应。

"用 30% 美债持仓做抵押，付清下个月所有 SaaS 订阅"——一次性任务，包含质押+支付两个步骤的组合执行。

"在保持税务效率的前提下做组合再平衡"——多目标优化任务，引擎自动选择不会触发短期资本利得的资产。

面向 AI 原生公司的金库即服务——CFO 只看月报，agent 全权打理。

AI 原生公司——不是用了 AI 的传统公司，是按 AI 能力重新设计组织架构的公司。典型画像：30 人以下，估值 1000 万-10 亿美元，主要收入来自全球客户的 USDT 或美元订阅，没有传统 CFO 团队，财务运营依赖少数几个 agent。

这类公司的痛点：没有银行愿意给他们开企业账户（业务模型太新，反洗钱风险评分高），同时资金又是全球流入流出的，传统私行的 corporate banking 服务慢得没法用。Oria 是为这个客群量身定做的。

收入模型

给股东的关键说明
这一层是 Oria 的第二增长曲线。AUM 模式（私行客户）的天花板是几十亿到几百亿美元，但 AI 原生公司 treasury 的市场规模随着 AI 渗透率指数级扩张——三年后可能比私行客户业务大一个数量级。把 Oria 早期就定位为"agent 友好的金融基础设施"，等 2028-2030 年 AI 原生公司爆发时，我们是默认的金融操作系统，不是后来才追赶的玩家。这是一次估值倍数的重置——市场不再按 fintech 估值（10-15x revenue），而按 infrastructure 估值（30-50x revenue）。

三大核心卖点

每一个都对应一个传统老钱过去解决不了的痛点。

01
USP · 信任革命
实时链上储备证明
FTX 之后传统资金对加密化资产最大的疑虑就是"资产真的在那儿吗"。Chainlink 实时上链 + 大所月度审计 + 独立董事签字，这个组合连私行都做不到——私行只给季度对账单。这是把"再看看"变成"开户"的临门一脚。
02
USP · 门槛下沉
机构资产门槛下沉
对冲基金 feeder 传统起投 50 万美元，私募信贷 100 万美元起。代币化把这些拆到 1-5 万美元一份，让 50 万-500 万美元资产的中产高净值（这个客群比超高净值大十倍）拿到原本拿不到的资产。
03
USP · 全球清算
全球清算无国界
USDT 不是匿名工具，是全球清算工具——它解决的是越南、巴西、土耳其、阿联酋的合格投资者无需先开美国银行账户就能配置美元资产。这个故事讲给监管也站得住：在帮新兴市场合格投资者获得本来就有权配置的美元资产。

必须明确的几件取舍

向股东讲清楚边界，比讲清楚卖点更重要。

完全匿名做不了，但对外不披露可以做
KYC 资料只对监管和法务保留，平台不向其他用户、不向公众披露任何客户身份与持仓。逻辑跟瑞士银行历史上的"银行保密法"一样：强 KYC + 强对外保密 = 现代私行模式。完全匿名是缉毒署的目标客户，不是我们的目标客户。
美国客户必须封禁，且需技术封禁
IP 屏蔽 + KYC 国籍核查 + 链上地址黑名单三层。SEC 对未注册证券发行的处罚是平台杀手，这一点不能让步。换来的是欧洲、亚洲、中东、拉美数十个司法辖区可以正常运营的可能。封禁美国 = 解锁全球其他市场。
综合费率会高于纯加密交易所
底层资产托管在传统金融体系，月度审计、托管、合规成本无法回避。但比起私行 1-2% 年管理费 + 进出场佣金，综合成本仍显著更低。我们不跟币安比费率，我们跟瑞银私行比费率——而且我们提供的资产配置丰富度，币安根本不做。
